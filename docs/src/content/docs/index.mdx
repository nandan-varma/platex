## What it is

`@nandan-varma/platex` compiles LaTeX to PDF from TypeScript. It works in any
framework that speaks the [Fetch API](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API)
— Next.js, TanStack Start, Astro, SvelteKit, Remix, Bun, Deno, Cloudflare
Workers — on Node.js or the edge. Nothing is hardcoded: every engine, timeout,
limit, auth token, and retry setting is an option with a sane default and an
env-var fallback.

```bash
npm install @nandan-varma/platex
```

  <TabItem label="One call">

```typescript
import { compile } from '@nandan-varma/platex';

const result = await compile(source, {
  serviceUrl: process.env.PLATEX_SERVICE_URL, // omit — read automatically
});

if (result.pdf) {
  // result.pdf is a Buffer — return, save, or stream it
}
```

  <TabItem label="Reusable client">

```typescript
// lib/platex.ts
import { createPlatexClient } from '@nandan-varma/platex';

export const platex = createPlatexClient({ timeout: 25_000, retry: 2 });

// anywhere
const result = await platex.compile(source);
```

  </TabItem>


  </TabItem>


## Why platex

<CardGrid>
  <Card title="Accurate PDF output" icon="document">
    Standard engine flags, multi-pass logic, and rerun heuristics for
    faithful LaTeX-to-PDF rendering — as accurate as your local TeX toolchain.
  </Card>
  <Card title="Runs anywhere" icon="rocket">
    Node.js or the edge. Three entry points: a full Node library, an
    edge-safe remote-only client, and a server factory to embed the API.
  </Card>
  <Card title="Two deployment targets" icon="puzzle">
    A remote service for Vercel (bundled Tectonic), or full TeX Live
    locally / self-hosted — the same library auto-selects.
  </Card>
  <Card title="Structured diagnostics" icon="list-format">
    Errors and warnings parsed from raw TeX logs, with file, line, and
    category — plus per-pass raw logs for debugging.
  </Card>
  <Card title="A real CLI" icon="seti:powershell">
    The same pipeline as a `platex` binary — watch mode, extra files,
    remote compilation, stdin, and JSON output.
  </Card>
  <Card title="Zero config" icon="setting">
    Set `PLATEX_SERVICE_URL` and go. Every option is optional, overridable
    per call, per client, or per deployment.
  </Card>
</CardGrid>

## Framework integrations

The same `handleCompileRequest` handler — a standard `Request` → `Response`
function — works in every framework. Next.js also gets a dedicated page per
rendering pattern:

- [Server Components](/rendering/ssr/) — `compile()` runs during the server render; the PDF ships in the initial HTML.
- [Client Components](/rendering/csr/) — edit LaTeX in the browser and compile on demand via `fetch`.
- [Server Actions](/rendering/server-actions/) — a form submits straight to a `'use server'` function.
- [Route Handlers](/rendering/route-handlers/) — the raw HTTP endpoint backing the other patterns.
- [Other frameworks](/frameworks/) — Astro, SvelteKit, Remix, TanStack Start, Bun, Deno, Hono, Cloudflare Workers.
