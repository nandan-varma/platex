## What it is

`@nandan-varma/platex` compiles LaTeX to PDF from TypeScript. The entire public
surface is one function — `compile()` — and every rendering pattern below calls
it the same way.

```bash
npm install @nandan-varma/platex
```

```typescript
import { compile } from '@nandan-varma/platex';

const result = await compile(source, {
  engine: 'pdflatex',
  serviceUrl: process.env.PLATEX_SERVICE_URL,
});

if (result.pdf) {
  // result.pdf is a Buffer
}
```

## Rendering patterns

The same `compile()` call works across every natural Next.js integration point.
Each pattern has its own page with a complete code sample:

- [Server Components](/rendering/ssr/) — `compile()` runs during the server render; the PDF ships in the initial HTML.
- [Client Components](/rendering/csr/) — edit LaTeX in the browser and compile on demand via `fetch`.
- [Server Actions](/rendering/server-actions/) — a form submits straight to a `'use server'` function.
- [Route Handlers](/rendering/route-handlers/) — the raw Node.js API route backing the other patterns.
